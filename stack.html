Stack

Stack adalah struktur data linear yang menggunakan konsep LIFO (Last In First Out), yaitu data terakhir yang masuk akan menjadi data pertama yang keluar.

Stack hanya memiliki satu pintu untuk proses penambahan dan penghapusan data, yaitu bagian TOP.

Ilustrasi Stack

Konsep stack mirip seperti tumpukan buku. Buku terakhir yang ditaruh di atas akan menjadi buku pertama yang diambil.

Karakteristik Stack

  • Menggunakan konsep LIFO
  • Penambahan data dilakukan di TOP
  • Penghapusan data dilakukan di TOP
  • Data paling atas paling mudah diakses

Operasi Pada Stack

  • Push → menambahkan data
  • Pop → menghapus data teratas
  • Peek → melihat data teratas
  • isEmpty → mengecek stack kosong
  • Size → menghitung jumlah data

Contoh Implementasi Stack Python 

def stack():
    s = []
    return s

def push(s, data):
    s.append(data)

def pop(s):
    data = s.pop()
    return data

def peek(s):
    return s[len(s)-1]

def isEmpty(s):
    return s == []

def size(s):
    return len(s)

# Program utama

data = stack()

push(data, 10)
push(data, 20)
push(data, 30)

print(data)

pop(data)

print(data)

Output Program 

[10, 20, 30]

[10, 20]

Penjelasan Coding

Fungsi push digunakan untuk menambahkan data ke dalam stack menggunakan append().

Fungsi pop digunakan untuk menghapus data paling atas menggunakan pop().

Fungsi peek digunakan untuk melihat elemen teratas tanpa menghapus data.

Study Kasus Stack

Stack sering digunakan pada fitur Undo dan Redo di aplikasi seperti Microsoft Word.

Ketika user mengetik sesuatu, data akan masuk ke stack. Saat tombol Undo ditekan, maka data terakhir akan dihapus terlebih dahulu.

Contoh Study Kasus Undo 

undo = []

undo.append("Ketik A")
undo.append("Ketik B")
undo.append("Ketik C")

print(undo)

undo.pop()

print(undo)

Output Study Kasus 

['Ketik A', 'Ketik B', 'Ketik C']

['Ketik A', 'Ketik B']

Kelebihan Stack

  • Struktur sederhana
  • Mudah diimplementasikan
  • Cepat untuk operasi push dan pop

Kekurangan Stack

  • Hanya bisa mengakses data teratas
  • Tidak fleksibel untuk pencarian data
